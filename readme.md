    <article class="markdown-body entry-content" itemprop="text"><ol>
<li>
<p>运行环境：python3，字符编码为unicode<br>
目前python2即使设置了'utf-8'，也会出现（__label_re中positon）一些问题，
可能会影响后期的过滤，</p>
</li>
<li>
<p>从database中读取kg_data没有写</p>
</li>
<li>
<p>character_level的几个问题</p>
</li>
<li>
<p>多线程终止自动<code>cat sample* sample</code></p>
</li>
<li>
<p>在sample.py中定义了两种过滤规则，一种是关键字，还有一种是长度，目前将这两个规则写在sample.py中</p>
</li>
<li>
<p>考虑到character_level，e1和e2的sent_id比较麻烦，目前不计划输入到模型中，（如果需要，可以在模型中单独为随机初始化embedding，但是没有意义，或者将embedding表示为所有word emb的average）</p>
</li>
<li>
<p>现在是所有的都要ner的，只是如果模型不需要ner，在生成数据时候就直接将ner的输入置为0</p>
</li>
<li>
<p>sample.py 的gen_train_data中最好生成一个基本版本，不需要每次都重复操作segment,ner,</p>
</li>
<li>
<p>多分类是不是多个类别最好不要强相关？</p>
</li>
</ol>
<ul>
<li>
<p>一些样本pattern：<br></p>
<ul>
<li>e1的董事长e2<br></li>
<li>e1董事长e2<br></li>
<li>e1的董事长候选人xx,e2,xx<br></li>
<li>e1的副总裁xx、e2、xx和xx<br></li>
<li>e1董事长xx,xx法定代表人e2  <em>（如果标记这种是负样本，算是正确的负样本了：
并列句子，且在kg中没有相应triple）</em><br></li>
<li>e1董事长，法定代表人e2<br></li>
<li>e1董事长、法定代表人e2<br></li>
<li>e1董事长兼法定代表人e2<br></li>
<li>e1......法定代表人e2<br></li>
<li>e1前总经理e2 <em>（如果kg中存储的未更新呢）</em><br></li>
<li>e1副总经理e2 <em>（如果kg已经存储的'总经理'或者'总监'呢）</em><br></li>
<li>e1担任e2总经理</li>
<li>e1曾担任e2总经理</li>
<li>e2...e1现任董事长，总经理，总裁</li>
</ul>
</li>
<li>
<p>从<strong>未知关系</strong>中找到相对质量高的负例</p>
<ol>
<li>如果句子被标记为'未知关系'(<code>rel == '未知关系'</code>) and <code>pos1-pos2 &gt; 50</code> ，可能介绍性质的出问题</li>
<li>并列句子( <code>sent.count(',') &gt; 6</code>) and <code>'，/、/；' in sent[pos1:pos2]</code></li>
<li>互斥关系作为负样本，（副总经理作为总经理的负样本）</li>
</ol>
</li>
<li>
<p>对正例稍作过滤</p>
<ol>
<li>rel in sent</li>
<li>time_related_words = ['曾经','前','曾','前任','候选人','',]</li>
<li>sensitive_word</li>
</ol>
</li>
<li>
<p><strong>vacab,embedding</strong></p>
<ol>
<li>如果用test.txt分词之后的进行处理，之后给模型的embeddings应该是加上'E1''E2'的，（
在训练的build_embeddings阶段，赋值为0，训练过程中得到的E1和E2保存下来，测试阶段，build_embeddings时候从保存的结果中加载二者的embeddings）
vocab.txt在训练emb的时候生成，在vocab.py的load_from_file中加上'E1'和'E2'</li>
<li>但是1训练的embeddings是不是要重新保存？只是保存特殊4个字符的，或者是所有的都重新保存？如果是都重新保存的话，预训练好像就不用做了；但是如果只是保存4个的，和其他word存在不同步问题？</li>
<li></li>
</ol>
</li>
<li>
<p>更改dataset.py</p>
<ol>
<li>将动态填充放在加载数据时</li>
<li>for index, row in content.iterrows():</li>
<li>rel = row.relations # 不会因为存储的顺序发生变化改变而发生错误</li>
<li>load_data ner2id = v.get_ner_id(ner) 写的很好</li>
<li></li>
</ol>
</li>
<li>
<p>文件名：</p>
<ol>
<li>clean后的文本名 original_text.txt</li>
<li>对 orginal_text.txt进行分词或者分字之后的 seg_original_text.txt</li>
<li></li>
</ol>
</li>
<li>
<p>执行顺序</p>
<ol>
<li>preprocess 原始的文本 clean训练集合</li>
<li>sample</li>
<li>train</li>
<li>preprocess predict.txt</li>
<li></li>
</ol>
</li>
</ul>
